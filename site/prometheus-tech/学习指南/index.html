    Prometheus 源码学习指南¶

    本指南旨在帮助你系统地学习 Prometheus 监控系统的架构设计与源码实现。

    一、项目概述¶

    Prometheus 是一个开源的系统和服务监控系统，由 CNCF 托管。其核心特性包括：

    • 多维度数据模型：通过指标名称和键值对标签定义时间序列
    • PromQL：强大的查询语言
    • Pull 模型：通过 HTTP 拉取方式收集指标
    • 服务发现：支持静态配置或动态服务发现
    • 本地存储：高效的时序数据库 (TSDB)
    • 告警与规则：支持告警规则和记录规则

    二、学习路径¶

    阶段 1：基础入门（1-2 周）¶

    1.1 理解核心概念¶

    阅读材料： - README.md - 项目概述和架构概览 - docs/getting_started.md - 入门教程 - docs/querying/basics.md - PromQL 基础 - docs/storage.md - 存储机制

    关键概念： - 时间序列（Time Series） - 指标（Metric）与标签（Labels） - 抓取（Scrape）与目标（Target） - 即时向量（Instant Vector）与范围向量（Range Vector） - 抓取间隔（Scrape Interval）与评估间隔（Evaluation Interval）

    1.2 实践操作¶

    # 1. 下载并运行 Prometheus
docker run --name prometheus -d -p 127.0.0.1:9090:9090 prom/prometheus

# 2. 配置自监控（参考 prometheus.yml 示例）
# 3. 使用 Expression Browser 查询指标
# 4. 尝试创建告警规则和记录规则

    练习： - 配置 Prometheus 监控自身 - 使用 PromQL 查询各种指标 - 创建简单的告警规则

    2.2 数据流理解¶

    写入流程： 

    Target → Scrape → Storage (TSDB) → Remote Write

    查询流程： 

    HTTP API → PromQL Engine → Storage (Local + Remote) → Response

    告警流程： 

    Rule Manager → PromQL → Notifier → Alertmanager

    阶段 3：核心模块深入学习（4-6 周）¶

    3.1 TSDB 存储引擎¶

    文档： - tsdb/README.md - tsdb/docs/format/README.md - 数据格式 - tsdb/docs/usage.md - 使用指南

    关键博客系列（强烈推荐）： 1. The Head Block 2. WAL and Checkpoint 3. Memory Mapping of Head Chunks from Disk 4. Persistent Block and its Index 5. Queries 6. Compaction and Retention 7. Snapshot on Shutdown

    源码阅读顺序： 

    1. tsdb/db.go              - TSDB 主入口
2. tsdb/head.go            - 内存 Head 块
3. tsdb/wlog/              - 预写日志
4. tsdb/block.go           - 持久化块
5. tsdb/compact.go         - 压缩逻辑

    关键知识点： - 2 小时块存储机制 - WAL（Write-Ahead Log） - 内存映射（mmap） - 压缩与保留策略 - 数据格式（chunks, index, tombstones）

    3.2 PromQL 查询引擎¶

    文档： - docs/querying/basics.md - docs/querying/operators.md - docs/querying/functions.md - docs/querying/api.md

    源码阅读顺序： 

    1. promql/parser/          - 词法与语法分析
2. promql/engine.go        - 查询执行引擎
3. promql/functions.go     - 内置函数实现
4. promql/operators.go     - 操作符实现

    关键知识点： - AST（抽象语法树）解析 - 查询执行计划 - 向量化执行 - 范围查询与即时查询 - 原生直方图支持

    3.3 服务发现与抓取¶

    文档： - discovery/README.md - docs/configuration/configuration.md

    源码阅读顺序： 

    1. discovery/manager.go    - 发现管理器
2. scrape/manager.go       - 抓取管理器
3. scrape/scrape.go        - 抓取循环
4. scrape/target.go        - 目标管理

    关键知识点： - 服务发现机制 - 目标重标签（relabeling） - 抓取超时与重试 - 指标重标签

    3.4 Web UI 与 API¶

    文档： - web/ui/README.md - 前端 UI 开发指南

    源码阅读顺序： 

    1. web/web.go              - Web 服务器
2. web/api/v1/api.go       - API v1 实现

    阶段 5：实践与贡献（持续）¶

    5.1 开发环境搭建¶

    # 1. 克隆仓库
git clone https://github.com/prometheus/prometheus.git
cd prometheus

# 2. 安装依赖
# Go: 版本见 go.mod
# NodeJS: 版本见 .nvmrc
# npm: >= 10

# 3. 构建
make build

# 4. 运行测试
make test

# 5. 代码检查
make lint

    5.2 贡献指南¶

    文档： - CONTRIBUTING.md - AGENTS.md - PR 提交规范

    贡献步骤： 1. 选择 low-hanging-fruit 标签的 issue 2. 在 issue 下评论认领 3. 创建分支并实现 4. 编写测试 5. 提交 PR（遵循规范格式）

    PR 规范： - 标题格式：area: short description - 包含 release-notes 块 - 每个 commit 独立可测试 - 使用 git commit -s 签名

    4.3 调试技巧¶

    1. 使用 promtool： 

      # 检查配置

# 检查规则

    六、常见问题¶

    Q1: 如何快速定位某个功能的代码？¶

    A: 1. 使用 internal_architecture.md 了解组件位置 2. 使用 GitHub 代码搜索 3. 查看相关测试文件

    Q2: TSDB 块是如何组织的？¶

    A: - 每 2 小时一个块 - 块包含 chunks（数据）、index（索引）、meta.json（元数据） - WAL 保证崩溃恢复 - 后台压缩成更大的块

    Q3: PromQL 查询是如何执行的？¶

    A: 1. 词法/语法分析生成 AST 2. 创建可执行查询 3. 从存储获取迭代器 4. 惰性执行表达式 5. 返回结果

    Q4: 如何参与贡献？¶

    A: 1. 阅读 CONTRIBUTING.md 2. 选择适合的 issue 3. 遵循 PR 规范 4. 确保测试通过

    七、学习建议¶

    1. 循序渐进：不要一开始就深入源码，先理解概念和使用
    2. 实践为主：多动手配置、查询、调试
    3. 阅读博客：TSDB 系列博客质量极高
    4. 查看测试：测试是最好的文档
    5. 参与社区：邮件列表和 IRC 是获取帮助的好地方
    6. 做笔记：记录关键理解和疑问
    7. 分享知识：写博客或做分享加深理解
