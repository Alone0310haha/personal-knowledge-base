    16 深分页为什么慢Scroll与SearchAfter怎么选

    【开场：共情与痛点】 小雅：【停顿1秒】老王，我有个特别丢人的线上故事。我们做列表翻页，用的还是那种“第 N 页”的方式，结果用户翻到后面直接卡成 PPT。面试官就问我：“深分页为什么慢？Scroll 和 Search After 怎么选？”我当时想说“加索引”，说出口又觉得不对。【笑】

    老王：害，这题不是你一个人踩过。深分页慢，很多时候不是“没有索引”，而是“你让系统做了大量白干的活”。【停顿1秒】咱把它讲成一个排队的画面，你就明白了。

    小雅：行，你说，但别讲太学术，我要那种我能复述给面试官的。【加重语气】

    【爆发：核心解析】 老王：from + size 的深分页[生活化类比：像你在奶茶店点单，非要说“把前面 1000 个顾客的订单都重新排一遍，然后再给我第 1001 个”]它的问题在于：每一页都要跳过前面大量结果，但 ES 不能凭空“跳过”，它得先把足够多的候选算出来，再丢掉前面的。

    小雅：【停顿1秒】这听起来像是浪费，但为什么浪费会这么夸张？

    老王：因为 ES 是分片并行的。每个分片都得返回更多候选给协调节点[生活化类比：像好几个窗口各自排队，你想拿到全场第 1001 号，得让每个窗口都把前面一大段人先报出来，最后在总台再合并排序]协调节点再全局排序，然后把你不要的前面 N 条丢掉。页数越深，丢掉的越多，浪费越大，内存和 CPU 都被拖住。能理解这个逻辑吗？

    小雅：【恍然大悟】所以不是“翻得越后面越慢”，而是“翻得越后面，白算的越多”。那怎么选 Scroll？

    老王：Scroll[生活化类比：像你要把仓库所有货物挨个盘点，管理员给你一张“盘点路线图”，你按这条路线一批批拿，不会每次都从仓库门口重新开始走]它适合“全量遍历/导出”这种离线任务。它会维护一个游标上下文，保证你拿到的是当时那一刻的快照式结果，方便你把全量数据走一遍。

    小雅：【停顿1秒】那它适合用户在线翻页吗？有时候产品也想用它省事。

    老王：一般不适合。因为它要保持上下文，占资源，而且用户翻页是随机的、可能中断的。[生活化类比：像你给每个顾客都开一个专属盘点路线，顾客走两步就跑了，路线还得一直保留着，仓库管理员会疯]所以 Scroll 更像“批处理工具”，不是“前台分页”。能理解这个逻辑吗？

    小雅：那 Search After 呢？它听着就像“从上一次继续”。

    老王：Search After[生活化类比：像你排队取号，上一页最后一个号码是 1050，那下一页你就从 1050 之后继续叫号，不用把前面 1049 个再点一遍]它适合在线翻页。你每次翻页都带着上一页最后一条的 sort 值，下一次从这个位置继续查。

    小雅：【停顿1秒】那面试官经常追：“为什么 search_after 需要唯一排序？”我当时答得很虚。

    老王：因为如果排序键不唯一，就可能出现重复或漏。[生活化类比：像你按“姓氏”排队叫号，很多人都姓王，你说“从王开始之后继续”，那到底从哪个王开始？]所以线上通常会用“时间 + _id”这种组合做稳定且唯一的排序，_id 当 tie-breaker。能理解这个逻辑吗？

    小雅：【加重语气】我插一句确认。【停顿1秒】所以 search_after 是“带游标但不保存服务器上下文”，而 scroll 是“服务器帮你保存上下文”，对吧？

    老王：对，你这个总结非常到位。

    【反转：面试官在想啥】 小雅：那面试官问这题，其实是在考我什么？是考我记不记得两个 API 名字吗？

    老王：生产里你确实不天天背 API，但面试官问你，其实是在考察你的“性能与访问模式意识”。你能不能区分在线分页和离线遍历，能不能解释深分页的浪费来自哪里，能不能选对工具。给你一句能背的面试话术。

    小雅：来一句能背的，把 from/size、scroll、search_after、唯一排序都讲进去。

    老王：你就说：“from + size 深分页会让每个分片计算并返回大量候选，协调节点全局排序后再丢弃前面的结果，页数越深浪费越大；全量遍历/导出用 scroll 维护游标上下文更合适，但不适合用户在线分页；在线翻页更推荐 search_after，依赖稳定且唯一的排序键，通常用时间加 _id 作为兜底避免重复和漏。”【停顿1秒】这句很稳。

    【收尾：三点总结】 小雅：【停顿1秒】我总结一下，其实就是三件事：第一件事儿，深分页慢是因为每次都要算出大量候选再丢掉，分片越多、页越深浪费越大；第二件事儿，Scroll 适合离线全量遍历，因为它要维护上下文，不适合前台用户翻页；第三件事儿，Search After 适合在线翻页，用上一页最后的排序值继续查，排序键要唯一，常用时间加 _id 防重复漏数据。【恍然大悟】这题我终于能讲清楚“为什么慢”和“怎么选”了。

    老王：对，就按“访问模式→成本→选型”讲，面试官听完基本就知道你写过线上分页了。
