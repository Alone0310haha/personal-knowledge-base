    13 QueryDSL分层term vs match query vs filter

    【开场：共情与痛点】 小雅：【停顿1秒】老王，我发现 ES 面试最容易把我问急的，就是 Query DSL。面试官问我：“你怎么分层理解 Query DSL？term 和 match 有啥区别？query 和 filter 又差在哪？”我当时一紧张，嘴里就剩“term 精确，match 模糊”。他说“那为什么你们线上把条件全放 must 里”，我瞬间感觉自己被看穿了。【笑】

    老王：害，这题问得好，说明他在看你会不会写“稳的查询”。【停顿1秒】你把 DSL 当成一棵树就行：每个节点都是一个条件，条件之间能组合。然后所有差异，最终都落到两件事：要不要分词、要不要算分。

    小雅：你先别上“树”，我怕我又晕。【加重语气】你先用最直观的方式讲 term 和 match。

    【爆发：核心解析】 老王：term[生活化类比：像对身份证号，必须一字不差才能认出人]是精确匹配，不分词。它适合 keyword、数字、日期这种字段，比如状态、租户 id、类目、时间范围。

    小雅：【停顿1秒】那 match 呢？

    老王：match[生活化类比：像你在通讯录里搜“老王”，系统会把你输入的词拆一拆、归一化一下，再去找可能相关的人]是全文检索，会走分析器分词，然后算相关性，最后给你个 _score。

    小雅：【恍然大悟】所以 term 是“拿原值对”，match 是“先处理再匹配”。那 query 和 filter 又怎么讲？我总觉得它俩都在“查”。

    老王：它俩都在查，但 filter[生活化类比：像进小区的门禁，符合就进，不符合就拦，根本不打分]不算 _score，更像业务硬条件，结果也更容易复用缓存。query[生活化类比：像相亲打分，条件匹配只是入场券，最后还要根据契合度排个顺序]会算分，用在“我想按相关性排一排”的场景。能理解这个逻辑吗？

    小雅：【停顿1秒】那我线上那种“所有条件都放 must”，是不是就等于很多本该是门禁的东西也拿去打分了？

    老王：对，这就会又慢又不稳定。你记一个稳的套路：硬条件放 filter，关键词检索放 must。[生活化类比：像你点外卖先过滤“只看 3km 内、评分 4.5 以上”，这属于门禁；然后再按“你想吃辣的、想吃面”这种偏好去排序]这样查询结构会很清晰：先筛掉不可能的，再在剩下的里算相关性。能理解这个逻辑吗？

    小雅：【加重语气】我插一句确认。【停顿1秒】也就是说 filter 负责缩小候选集合，query 负责在候选里排序，对吧？

    老王：对，完全正确。

    小雅：那还有一个我总被追的点：“为什么 filter 更快？”我怕我答成“因为缓存”太敷衍。

    老王：你可以这么说：filter 不算分，执行路径更短；而且很多 filter 条件在不同请求里复用度高，更容易被缓存命中。[生活化类比：门禁刷卡这种动作每天重复，系统当然能优化得很快；相亲打分每个人都不一样，没法复用]你这样答就不空。能理解这个逻辑吗？

    小雅：match_phrase 呢？有时候面试官顺手一带。

    老王：match_phrase[生活化类比：像你找一段台词，要求词序基本一致，不能把“我爱你”搜成“你爱我”]是短语匹配，词序和距离更严格。你提一句“更严格、可以放宽距离”就够了，别展开到像念文档。能理解这个逻辑吗？

    【反转：面试官在想啥】 小雅：所以这题面试官到底想考啥？真的是考我记不记得哪些 DSL 吗？

    老王：生产里你确实不天天背 DSL 名字，但面试官问你，其实是在考察你的“查询建模意识”。你能不能把业务条件分成硬过滤和相关性检索，写出稳定、可解释、性能可控的查询结构。给你一句能背的面试话术。

    小雅：来一句能背的，我要那种能把 term/match 和 query/filter 一口气串起来的。

    老王：你就说：“我会把 DSL 当成可组合的查询树，term 类查询用于 keyword/数值/日期 的精确过滤，match 类查询用于 text 字段的分词检索与相关性；业务硬条件放在 filter 里不参与评分且更易缓存，关键词检索放在 must 里参与评分，形成先过滤再按相关性排序的稳定结构。”【停顿1秒】这句非常面试友好。

    【收尾：三点总结】 小雅：【停顿1秒】我总结一下，其实就是三件事：第一件事儿，term 不分词走精确匹配，match 会分词并算相关性；第二件事儿，filter 是门禁不打分更稳，query 是打分排序更贵；第三件事儿，线上常用结构是先用 filter 把硬条件卡住，再用 must 做关键词检索，性能和结果都更可控。【恍然大悟】我感觉我终于能解释“为什么这么写”了。

    老王：对，面试官要的就是“你写得稳”，而不是“你背得熟”。这题你按这个讲法，基本就拿下了。
