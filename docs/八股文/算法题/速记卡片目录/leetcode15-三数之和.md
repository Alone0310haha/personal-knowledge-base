## 题目基本信息

- 题号与题名：15. 三数之和
- 难度：中等
- 题型分类：双指针

---

## 数据范围与复杂度分析

- 题目数据范围：`3 <= nums.length <= 3000`，`-10^5 <= nums[i] <= 10^5`
- 允许的时间复杂度上限：O(n²)（n=3000，O(n²)=9×10^6，可接受；O(n³) 超时）
- 本题实际复杂度：时间 O(n²) / 空间 O(1)（不计输出）

---

## 关键词识别

| 题目中的关键词或约束 | 暗示使用的方法或结构 |
|---|---|
| 三元组之和为 0 | 固定一个数，转化为两数之和 |
| 不可包含重复三元组 | 排序后跳过相邻重复元素 |
| 数组无序，需要枚举组合 | 先排序，再双指针收缩 |

---

## 触发条件（一句话）

「有序数组 + 多数之和为定值 + 去重 -> 排序 + 双指针」

---

## 解法骨架（关键动作，不是完整代码）

1. **排序**：`Arrays.sort(nums)`，为双指针和去重提供基础
2. **枚举最小数**：外层 `for` 遍历 `index`，若 `nums[index] > 0` 可直接 break（排序后后面只会更大）
3. **去重外层**：`index > 0 && nums[index] == nums[index-1]` 则跳过
4. **双指针收缩**：`left = index+1`，`right = n-1`；三数之和 > 0 则 `right--`，< 0 则 `left++`，== 0 则记录并同时移动两指针，移动时跳过相同元素去重

---

## 容易出错的边界

- **外层去重时机**：判断 `index > 0`（而非 `index >= 0`），避免把第一个元素直接跳过
- **内层去重时机**：`left > index + 1`（而非 `left > 0`），因为 `left` 从 `index+1` 开始，不能把起点本身跳过
- **left == right 时 break**：找到匹配后移动双指针时要保证 `left < right`，否则两指针重合越界
- **`nums[index] > 0` 提前终止**：排序后最小数已 > 0，三数之和必然 > 0，可剪枝

---

## 一句话复习总结

排序后固定最左数，双指针从两端夹逼，三处去重（外层枚举、left、right）缺一不可。
