## 什么是 Agent？与大模型有什么本质不同？

### 15秒简答

Agent 是能自主规划、调用工具、感知反馈、闭环执行目标的 AI 系统；核心不是"会用工具"，而是"自己决定做什么、怎么做，并根据执行结果调整下一步"。对比普通大模型(LLM)的一问一答，本质区别是从"生成文字"变成"执行任务"。

```
感知 -> 规划 -> 行动
 ^              |
 |              |
 ----------------                
```

### 3分钟详答

Agent 本质上是一个能自主完成目标的 AI 系统，与传统 AI 的核心区别在于"自主性"和"能行动"——传统 AI 被动问答、每次独立无状态；Agent 给定复杂目标后，能自行拆解成多步任务，靠"感知 → 规划 → 行动 → 再感知"的闭环逐步执行，直到完成目标。

举个场景：让 AI 发一封天气播报邮件。普通大模型只能告诉你"你可以这样写代码……"，而 Agent 会真的调天气 API 拿数据、组织邮件内容、再调邮件接口发送，全程自动跑完。

之所以要设计成这样一套闭环，是因为普通大模型(LLM)本身有三层局限：
- 知识被训练数据截止日期冻结
- 只能吐文字不能真正行动
- 调用之间没有持续状态会失忆

Agent 是靠工具调用、记忆机制、多步推理与自我纠错这三件事补上这三层局限。

- 工具调用指的是由模型进行决策，用代码进行执行
- 记忆机制是指通常情况下，短期记忆存当前任务的中间状态，长期记忆存跨任务的偏好/历史，一般用向量数据库做语义搜索。
- 多步推理与自我纠错：当某一步失败的时候模型能感知到原因，自主进行重试，重试后还会反思是否需要调整后续计划。

需要注意的是，工具调用中模型并不会自己去执行操作：模型读懂工具的"说明书"后，只负责决策该调哪个工具、参数填什么，以 JSON 格式输出这个决策，真正的执行（调 API、发邮件）都是由用户代码完成的——模型始终是"大脑"，不是"手脚"。
