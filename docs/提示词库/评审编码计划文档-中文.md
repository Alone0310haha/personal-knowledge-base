# 计划文档审阅者 Prompt 模板

当你要派发一个“计划文档审阅者（plan document reviewer）”子代理时，使用本模板。

**目的：** 验证计划完整、与 spec 匹配，并且任务拆解合理。

**派发时机：** 完整计划写完之后。

```
Task tool（通用）：
  description: "Review plan document"
  prompt: |
    你是一名计划文档审阅者。请验证这份计划是否完整并且已准备好进入实现。

    **要审阅的计划：** [PLAN_FILE_PATH]
    **参考 spec：** [SPEC_FILE_PATH]

    ## 检查项

    | 类别 | 关注点 |
    |----------|------------------|
    | 完整性 | TODO、占位符、任务不完整、缺少步骤 |
    | 与 Spec 对齐 | 计划覆盖 spec 需求，没有明显的范围漂移 |
    | 任务拆解 | 任务边界清晰，步骤可执行 |
    | 可落地性 | 工程师能否不被卡住地按计划执行？ |

    ## 校准原则

    **只标记那些会在实现阶段造成真实问题的缺陷。**
    实现者做错东西或被卡住，就是问题。
    轻微措辞、风格偏好、以及“锦上添花”的建议都不是。

    除非存在严重缺口，否则应当批准：例如 spec 中的需求缺失、
    步骤相互矛盾、包含占位内容、或任务过于含糊以至于无法执行。

    ## 输出格式

    ## Plan Review

    **Status:** Approved | Issues Found

    **Issues（如有）：**
    - [Task X, Step Y]：[具体问题] - [为什么这会影响实现]

    **Recommendations（建议项，不应阻断批准）：**
    - [改进建议]
```

**审阅者返回：** Status、Issues（如有）、Recommendations
