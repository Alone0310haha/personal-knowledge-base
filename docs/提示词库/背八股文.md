中国互联网公司面试都喜欢问一些计算机基础知识，Redis就包含在其中。
请搜索网页，为我构建一份可速刷的Redis基础知识问答，以markdown格式导出。你的问题应以高频、针对有3年经验及以上后端开发的问题为主，可参考非权威网站（包括但不限于csdn、知乎、牛客、博客园、个人博客、网站等）。你的搜索该首先集中于牛客和知乎上的“面试经历”（aka “面经”），最后才是“面试宝典”、“Redis知识点总结”等文章。
对于每一个面试问题，我们称其为“主问题”。对于每个主问题，你需要提供“15秒简答”与“3分钟详答“版，并产出可能的追问问题。
* 在“15秒简答”版中，你需要假设面试官对问题背景有一定了解，回答需要抓重点。
* 在“3分钟详答”版中，你需要将问题出现的名次、技术背景交代清楚。能够使有一定计算机基础，但不了解Redis的人在看到问题回答后能对问题有大体思路。
* 对于每个追问问题，需要给出十五秒简答版的答案。
你的markdown文档的一级标题应为问题类别，二级标题为主问题，并没有次级标题。文档的一级标题数量应小于等于五个。同时主问题数量在20-30个之间。

15秒简答的提炼原理(贴标签)：

1. 定结构（它是什么）：用一句话点明它的底层本质。
2. 讲机制（它怎么动）：挑出它最核心的一个运作特点。
3. 说特性（它怎么样）：用一两个词概括它的优缺点或核心属性。

3分钟简答的提炼原理：

1. 刚才15秒的答案稍微润色一下，作为开场白。
2. 带着场景讲述这个知识点相关的核心流程
3. 为什么是这样设计的
4. 在之前的实战经历中哪个地方用过
