# Spring Boot 配置敏感信息如何加密？

[Spring Boot 配置敏感信息如何加密？](https://www.bilibili.com/video/BV1qb6EB2EGi?spm_id_from=333.788.videopod.episodes&vd_source=c7bea6fb9f92988f548051c4ed7de1e7&p=43)

## 讨论视角（这期在解决什么）

- 从“Spring Boot 配置文件里的敏感信息（DB/Redis 密码、远程地址）如何防止泄露”这一面试题出发，强调配置泄露的真实风险（无权限人员获取、服务器被入侵后被进一步利用）
- 把问题拆成两件事：如何生成密文、运行时如何解密，使应用仍能正常启动连接数据库/Redis
- 采用现成轮子 Jasypt：配置里写 `ENC(...)` 密文，启动时提供解密密钥完成透明解密

## 核心论点与分论点

### 核心论点

Spring Boot 配置文件的敏感信息可以通过 Jasypt 进行“配置值加密 + 运行时解密”：把明文（用户名、密码、URL 等）替换为 `ENC(密文)`，构建/交付时密文进入配置文件；应用启动时通过命令行参数提供加密密钥（视频称“私钥”），由 Jasypt 自动解密还原成真实配置值，从而降低配置泄露后的直接危害。

### 分论点 1：为什么要加密配置敏感信息

- 配置里常见敏感项：数据库用户名/密码、Redis 地址/密码、远程服务地址等
- 风险场景：
  - 无权限的人拿到配置文件后恶意利用
  - 服务器被入侵后，攻击者可继续横向到 DB/Redis 等核心基础设施
- 加密后的收益：即使配置文件被拿到，也不能直接得到明文凭据

### 分论点 2：加密方案不需要自研，直接使用 Jasypt

- 依赖引入后即可对配置值进行加密/解密
- 密文写法：`ENC(....)`（视频演示生成的格式）

### 分论点 3：密文如何生成：用 Maven 插件/命令行生成 `ENC(...)`

- 视频给出更便捷方式：引入 Jasypt 的 Maven 插件后，通过命令行执行插件生成密文
- 生成时关键参数（视频口径）：
  - `password`：用于加密/解密的密钥（“私钥”，只有开发者掌握）
  - `value`：要加密的明文内容（例如数据库密码、用户名、URL）
- 生成结果：输出 `ENC(密文)`，复制粘贴到 `application.yml` 对应配置项

### 分论点 4：运行时如何解密：启动时提供密钥，否则会因密文导致启动报错

- 未配置解密密钥时：应用直接读取到密文，无法完成连接等初始化，可能启动报错
- 解决：启动时通过命令行参数提供密钥
  - 本地运行：在启动参数中设置 Jasypt 的密钥
  - 服务器部署：`java -jar` 启动时同样追加该参数
- 结果：应用启动时拿到密钥自动解密配置值，运行正常

### 分论点 5：落地注意点：密钥不要写回配置文件

- 加密只能降低“配置泄露”风险，密钥一旦被同样泄露，加密形同虚设
- 工程习惯：密钥应通过启动参数、环境变量或更安全的密钥管理方式注入

## 关键数据、案例与结论（逐条可复述）

- 加密对象：数据库用户名/密码、数据库连接地址、Redis 地址/密码等
- 密文格式：`ENC(....)`
- 关键结论：配置加密的核心是“密文进配置、密钥在运行时注入”，否则无法解密也无法启动
- 延伸问题（视频收尾）：如果要“加密 jar 内类文件防反编译”，属于另一类保护，需要不同工具链（与配置加密不同）
