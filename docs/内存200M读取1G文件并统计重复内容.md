# 内存200M读取1G文件并统计重复内容

[内存200M读取1G文件并统计重复内容](https://www.bilibili.com/video/BV1qb6EB2EGi?spm_id_from=333.788.videopod.episodes&vd_source=c7bea6fb9f92988f548051c4ed7de1e7&p=16)

## 讨论视角

- 以“面试场景题”的角度讨论：内存只有 200MB，如何读取 1GB 文件，并统计内容重复次数
- 核心关注点是：不能一次性读入内存；在“内容高度不重复”的极端情况下，内存统计结构会爆掉

## 核心论点与分论点

### 核心论点

处理“内存放不下的大文件统计”本质是两件事：流式分块读取 + 必要时把中间结果外部化（落盘分治），再逐块统计汇总。

### 分论点 1：理想情况（内存能装下统计结构）

- 用缓冲区分块读取文件（例如每次 1MB；内存 200MB 时，每次 10MB/50MB 也可）
- 对读取到的内容循环处理，用 Map 计数：key 为“内容单元”，value 为出现次数

### 分论点 2：极端情况（几乎全不重复导致 Map OOM）

- 问题根因：1GB 内容几乎全不重复时，Map 会产生大量 key，最终撑爆应用内存
- 解决手段：把“内存放不下的 key”分流到磁盘
  - 对读到的内容做哈希并取模（示例：mod 1024），得到 1~1024 的分片编号
  - 将该内容写入对应分片文件（示例：字符“徐”取模后落到 3 号文件，就写入 3 号分片文件）
- 第二轮：逐个读取分片文件
  - 对当前分片文件分块读取，用 Map 统计该分片内的重复次数并输出
  - 处理完一个分片就清空 Map，再处理下一个分片，直到全部完成

## 关键数据 / 案例 / 结论

- 约束：200MB 内存处理 1GB 文件
- 读取方式：缓冲区分块读（示例提到 1MB；也可 10MB/50MB，取决于内存余量）
- 分治参数：哈希取模 1024，将数据分散到 1024 个分片文件
- 结论：当“统计结构可能超内存”时，通过“先分片落盘、再逐片统计”的外部化分治方式可完成统计
