## 2.1 安装方式对比

Prometheus 支持多种安装方式，每种方式适用于不同的场景：

| 安装方式 | 适用场景 | 优点 | 缺点 |
|----------|----------|------|------|
| **Docker** | 快速体验、测试环境 | 简单快捷、环境隔离 | 需要 Docker 环境 |
| **二进制** | 生产环境、长期使用 | 性能好、易管理 | 需要手动配置 |
| **源码编译** | 开发、定制需求 | 可定制、最新特性 | 耗时长、依赖多 |

**推荐**：初学者使用 Docker，生产环境使用二进制包。

## 2.2 方式一：Docker 安装（推荐初学者）

### 2.2.1 前置要求

- 已安装 Docker（版本 20.10+）
- 确保 9090 端口未被占用
- 至少 1GB 可用磁盘空间

### 2.2.2 验证 Docker 安装

```bash
# Windows PowerShell 或 CMD
docker --version

# 输出示例：
# Docker version 24.0.7, build afdd53b
```

如果未安装 Docker，请访问 https://docs.docker.com/get-docker/ 下载安装。

### 2.2.3 启动 Prometheus 容器

```bash
docker run -d \
  --name prometheus \
  -p 127.0.0.1:9090:9090 \
  prom/prometheus:latest
```

**参数说明**：
- `-d`：后台运行容器
- `--name prometheus`：容器命名为 prometheus
- `-p 127.0.0.1:9090:9090`：端口映射（仅本地访问，更安全）
- `prom/prometheus:latest`：使用最新版本的官方镜像

**安全提示**：使用 `127.0.0.1:9090` 而不是 `9090`，这样只有本机可以访问，避免暴露到外网。

### 2.2.4 验证容器运行

```bash
# 查看容器状态
docker ps | grep prometheus

# 输出示例：
# CONTAINER ID   IMAGE                STATUS          PORTS                    NAMES
# abc123def456   prom/prometheus      Up 2 minutes    127.0.0.1:9090->9090/tcp prometheus
```

### 2.2.5 访问 Prometheus

打开浏览器访问：http://localhost:9090

你应该能看到 Prometheus 的 Web UI 界面。

### 2.2.6 挂载配置文件（可选）

如果想自定义配置，可以挂载配置文件：

```bash
# 1. 创建配置目录
mkdir -p C:\prometheus\config

# 2. 创建配置文件 prometheus.yml
# 内容见 2.4 节

# 3. 启动容器时挂载配置
docker run -d \
  --name prometheus \
  -p 127.0.0.1:9090:9090 \
  -v C:\prometheus\config\prometheus.yml:/etc/prometheus/prometheus.yml \
  prom/prometheus:latest
```

### 2.2.7 停止和删除容器

```bash
# 停止容器
docker stop prometheus

# 启动容器
docker start prometheus

# 删除容器（会丢失数据）
docker rm prometheus
```

## 2.3 方式二：二进制包安装（生产环境推荐）

### 2.3.1 下载二进制包

**方式 1：从官网下载**

访问：https://prometheus.io/download/

选择适合你系统的版本：
- Windows: `prometheus-3.x.x.windows-amd64.zip`
- Linux: `prometheus-3.x.x.linux-amd64.tar.gz`
- macOS: `prometheus-3.x.x.darwin-amd64.tar.gz`

**方式 2：使用 PowerShell 下载（Windows）**

```powershell
# 下载最新版本（请替换为实际最新版本号）
$url = "https://github.com/prometheus/prometheus/releases/download/v3.0.0/prometheus-3.0.0.windows-amd64.zip"
$output = "C:\temp\prometheus.zip"

Invoke-WebRequest -Uri $url -OutFile $output

# 解压
Expand-Archive -Path $output -DestinationPath "C:\prometheus"
```

**方式 3：使用 wget 下载（Linux/macOS）**

```bash
# 下载
cd /opt
wget https://github.com/prometheus/prometheus/releases/download/v3.0.0/prometheus-3.0.0.linux-amd64.tar.gz

# 解压
tar -xvf prometheus-3.0.0.linux-amd64.tar.gz
cd prometheus-3.0.0.linux-amd64
```

### 2.3.2 目录结构

解压后的目录结构：

```
prometheus-3.0.0/
├── prometheus.exe          # Prometheus 主程序（Windows）
├── prometheus              # Prometheus 主程序（Linux/macOS）
├── promtool.exe            # 配置检查工具（Windows）
├── promtool                # 配置检查工具（Linux/macOS）
├── console_libraries/      # 控制台模板库
├── consoles/               # 控制台模板
├── LICENSE                 # 许可证
├── NOTICE                  # 声明
└── prometheus.yml          # 示例配置文件
```

### 2.3.3 启动 Prometheus

**Windows**：
```powershell
cd C:\prometheus\prometheus-3.0.0.windows-amd64
.\prometheus.exe --config.file=prometheus.yml
```

**Linux/macOS**：
```bash
cd /opt/prometheus-3.0.0.linux-amd64
./prometheus --config.file=prometheus.yml
```

**输出示例**：
```
ts=2024-01-01T10:00:00.000Z caller=main.go:534 level=info msg="Starting Prometheus Server" version="(version=3.0.0)"
ts=2024-01-01T10:00:00.001Z caller=main.go:539 level=info msg="operational information" build_context="(go=go1.21.0)"

ts=2024-01-01T10:00:00.005Z caller=main.go:570 level=info msg="Server is ready to receive web requests."
```

看到 "Server is ready" 表示启动成功！

### 2.3.4 后台运行（Linux）

```bash
# 使用 nohup 后台运行
nohup ./prometheus --config.file=prometheus.yml > prometheus.log 2>&1 &

# 或使用 systemd 服务（推荐）
```

## 2.4 配置 Prometheus 监控自身

### 2.4.1 理解配置文件

Prometheus 的配置文件是 YAML 格式，默认文件名为 `prometheus.yml`。

**核心配置项**：
- `global`：全局配置
- `scrape_configs`：抓取配置
- `rule_files`：规则文件
- `alerting`：告警配置
- `remote_write`：远程写入
- `remote_read`：远程读取

### 2.4.2 创建自监控配置

**文件位置**：
- Docker: `C:\prometheus\config\prometheus.yml`
- 二进制：`prometheus-3.0.0\prometheus.yml`

**配置内容**：
```yaml
# prometheus.yml - Prometheus 自监控配置

# 全局配置
global:
  # 抓取间隔：默认 15 秒
  scrape_interval: 15s
  
  # 评估规则间隔：默认 15 秒
  evaluation_interval: 15s
  
  # 附加到所有时间序列的外部标签
  external_labels:
    monitor: 'prometheus-tutorial'
    environment: 'development'

# 告警配置（可选，后续课程会详细讲解）
alerting:
  alertmanagers:
    - static_configs:
        - targets: []

# 规则文件（可选，后续课程会详细讲解）
rule_files:
  # - "first_rules.yml"

# 抓取配置
scrape_configs:
  # 第一个抓取任务：Prometheus 自身
  - job_name: 'prometheus'
    
    # 覆盖全局抓取间隔，5 秒抓取一次
    scrape_interval: 5s
    
    # 静态配置：手动指定目标
    static_configs:
      # 监控 localhost:9090 的 Prometheus
      - targets: ['localhost:9090']
        # 添加自定义标签
        labels:
          group: 'monitoring'
          team: 'devops'
```

### 2.4.3 配置说明

**全局配置（global）**：
```yaml
global:
  scrape_interval: 15s      # 默认抓取间隔
  evaluation_interval: 15s  # 规则评估间隔
  external_labels:          # 外部标签（用于联邦或远程存储）
    monitor: 'prometheus-tutorial'
```

**抓取配置（scrape_configs）**：
```yaml
scrape_configs:
  - job_name: 'prometheus'  # 任务名称（会成为 job 标签）
    scrape_interval: 5s     # 覆盖全局抓取间隔
    static_configs:
      - targets: ['localhost:9090']  # 目标地址
        labels:
          group: 'monitoring'        # 自定义标签
```

### 2.4.4 重启 Prometheus 使配置生效

**Docker 方式**：
```bash
# 停止并删除旧容器
docker stop prometheus
docker rm prometheus

# 使用新配置启动
docker run -d \
  --name prometheus \
  -p 127.0.0.1:9090:9090 \
  -v C:\prometheus\config\prometheus.yml:/etc/prometheus/prometheus.yml \
  prom/prometheus:latest
```

**二进制方式**：
```bash
# 方式 1：重启进程
# 先停止（Ctrl+C），再重新启动

# 方式 2：热重载配置（推荐）
# Windows: 需要重启进程
# Linux: 发送 SIGHUP 信号
kill -SIGHUP <prometheus_pid>

# 或通过 HTTP API
curl -X POST http://localhost:9090/-/reload
```

### 2.4.5 验证配置

访问：http://localhost:9090/targets

你应该能看到一个目标：
```
Endpoint        State    Labels
localhost:9090  UP       job="prometheus" group="monitoring" team="devops"
```

**状态说明**：
- **UP**：目标健康，正常抓取
- **DOWN**：目标不可达
- **Unknown**：未知状态

## 2.6 验证 Prometheus 服务指标

### 2.6.1 访问 Metrics 端点

Prometheus 暴露自身指标在 `/metrics` 端点：

```bash
# 浏览器访问
http://localhost:9090/metrics

# 或使用 curl
curl http://localhost:9090/metrics
```

**输出示例**：
```
# HELP prometheus_target_interval_length_seconds Actual intervals between scrapes.
# TYPE prometheus_target_interval_length_seconds summary
prometheus_target_interval_length_seconds{quantile="0.5"} 5.001
prometheus_target_interval_length_seconds{quantile="0.9"} 5.002
prometheus_target_interval_length_seconds{quantile="0.99"} 5.003
prometheus_target_interval_length_seconds_count 120
prometheus_target_interval_length_seconds_sum 600.12
```

### 2.6.2 理解 Metrics 格式

```
# HELP <指标名称> <描述>
# TYPE <指标名称> <类型>
<指标名称>{标签="值"} 数值 时间戳
```

**示例解析**：
```
# HELP prometheus_target_interval_length_seconds Actual intervals between scrapes.
# TYPE prometheus_target_interval_length_seconds summary
prometheus_target_interval_length_seconds{quantile="0.5"} 5.001
```

- **HELP**：指标描述
- **TYPE**：指标类型（summary）
- **指标行**：指标名称 + 标签 + 值

### 2.6.3 重要指标说明

| 指标名称 | 含义 | 类型 |
|----------|------|------|
| `prometheus_target_interval_length_seconds` | 实际抓取间隔 | Summary |
| `prometheus_target_scrape_pool_sync_total` | 抓取池同步次数 | Counter |
| `prometheus_sd_discovered_targets` | 服务发现的目标数 | Gauge |
| `prometheus_tsdb_head_samples_appended_total` | 追加到 TSDB 的样本数 | Counter |
| `prometheus_http_requests_total` | HTTP 请求总数 | Counter |
| `prometheus_engine_queries` | 查询执行次数 | Counter |

## 2.8 常见问题与排查

### 问题 2.8.1：端口被占用

**错误信息**：
```
Error: listen tcp :9090: bind: Only one usage of each socket address is allowed
```

**原因**：9090 端口已被其他程序占用

**解决方案**：
```bash
# 方案 1：查找并关闭占用端口的程序
# Windows
netstat -ano | findstr :9090
taskkill /PID <PID> /F

# Linux
lsof -i :9090
kill -9 <PID>

# 方案 2：修改 Prometheus 端口
./prometheus --web.listen-address=":9091"
```

### 问题 2.8.2：容器无法启动

**错误信息**：
```
docker: Error response from daemon: port 9090 is already allocated
```

**解决方案**：
```bash
# 检查是否有同名容器
docker ps -a | grep prometheus

# 删除旧容器
docker rm -f prometheus

# 重新启动
docker run -d --name prometheus -p 127.0.0.1:9090:9090 prom/prometheus
```

### 问题 2.8.3：配置文件错误

**错误信息**：
```
Error loading config: one or more errors occurred while applying the new configuration
```

**排查步骤**：
```bash
# 使用 promtool 检查配置
./promtool check config prometheus.yml

# 输出示例：
# SUCCESS: 1 rule files found
# SUCCESS: prometheus.yml is valid prometheus config file syntax
```

**常见错误**：
- YAML 缩进错误（必须使用空格，不能用 Tab）
- 冒号后缺少空格
- 拼写错误

### 问题 2.8.4：目标状态为 DOWN

**可能原因**：
1. 目标地址错误
2. 网络不通
3. 目标服务未启动
4. 防火墙阻止

**排查步骤**：
```bash
# 1. 检查目标是否可达
curl http://localhost:9090/metrics

# 2. 检查网络连通性
telnet localhost 9090

# 3. 查看 Prometheus 日志
docker logs prometheus
# 或
tail -f prometheus.log

# 4. 检查防火墙
# Windows: 关闭防火墙测试
# Linux: sudo ufw status
```

### 问题 2.8.5：内存占用过高

**现象**：Prometheus 进程占用大量内存

**原因**：
- 时间序列过多
- 抓取间隔太短
- 保留时间太长

**解决方案**：
```bash
# 调整启动参数
./prometheus \
  --storage.tsdb.retention.time=15d \
  --storage.tsdb.retention.size=10GB
```

**参数说明**：
- `--storage.tsdb.retention.time`：数据保留时间（默认 15d）
- `--storage.tsdb.retention.size`：数据保留大小（默认无限制）

5. 可以使用 promtool 检查配置文件语法。（ ）

### 实操题

1. 使用 Docker 启动一个 Prometheus 实例，并配置其监控自身。
2. 访问 Targets 页面，截图证明目标状态为 UP。
