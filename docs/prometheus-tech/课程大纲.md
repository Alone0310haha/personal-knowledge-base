# Prometheus 系统学习课程大纲（30 节课）

本课程大纲旨在帮助你从零开始系统学习 Prometheus 监控系统，涵盖使用、架构、集成开发和自主实现四个维度。

---

## 第一阶段：基础入门与核心概念（第 1-6 课）

### 第 1 课：Prometheus 概述与监控理念
**目标**：理解 Prometheus 是什么、为什么选择它、核心特性
- 监控系统的发展与 Prometheus 的定位
- Prometheus 的核心特性（Pull 模型、多维度数据、PromQL 等）
- Prometheus 与其他监控系统的对比
- 典型应用场景
- **实践**：阅读官方文档和 README，了解项目背景

### 第 2 课：快速上手 - 安装与运行
**目标**：能够在本地运行 Prometheus 实例
- 安装方式对比（Docker、二进制、源码编译）
- 目录结构与核心文件
- 启动 Prometheus 并验证
- 访问 Web UI 和状态页面
- **实践**：使用 Docker 运行 Prometheus，访问 localhost:9090

### 第 3 课：配置文件详解
**目标**：掌握 Prometheus 配置文件的结构与语法
- 全局配置（global）
- 抓取配置（scrape_configs）
- 规则文件配置（rule_files）
- 配置验证与热重载
- **实践**：配置 Prometheus 监控自身，使用 SIGHUP 热重载配置

### 第 4 课：时间序列数据模型
**目标**：深入理解 Prometheus 的数据模型
- 指标（Metric）与时间序列（Time Series）
- 标签（Labels）的作用与规范
- 指标类型（Counter、Gauge、Histogram、Summary）
- 自动生成的标签（job、instance）
- **实践**：查询不同类型的指标，理解标签过滤

### 第 5 课：PromQL 基础查询
**目标**：掌握 PromQL 的基本语法
- 即时向量与范围向量
- 基本查询语法
- 标签匹配操作符
- 聚合操作
- **实践**：使用 Expression Browser 执行各种查询

### 第 6 课：PromQL 进阶操作
**目标**：掌握 PromQL 的高级功能
- 数学运算与比较操作
- 时间范围函数（rate、irate、increase）
- 聚合函数（sum、avg、max、min）
- 子查询
- **实践**：编写复杂的查询表达式，绘制图表

---

## 第二阶段：架构理解与数据流（第 7-12 课）

### 第 7 课：Prometheus 整体架构
**目标**：理解 Prometheus 的组件构成与协作
- 架构图解析（Retrieval、Storage、HTTP Server、Rule Manager、Notifier）
- 各组件职责
- 组件间通信机制
- **实践**：绘制架构图，标注各组件职责

### 第 8 课：服务发现机制（Service Discovery）
**目标**：理解服务发现的原理与实现
- 服务发现的概念与作用
- 支持的服务发现类型
- Discovery Manager 的工作原理
- 目标组（Target Group）更新机制
- **实践**：配置文件服务发现（file_sd），观察目标变化

### 第 9 课：抓取管理（Scrape Manager）
**目标**：理解指标抓取的完整流程
- Scrape Manager 与 Scrape Pool
- 抓取循环（Scrape Loop）
- 目标标签与重标签（Relabeling）
- 抓取超时与重试机制
- **实践**：配置目标重标签和指标重标签

### 第 10 课：存储子系统 - TSDB 概述
**目标**：理解 TSDB 的基本概念
- TSDB 的核心特性
- 数据块（Block）组织
- 内存 Head Block 与持久化 Block
- WAL（Write-Ahead Log）的作用
- **实践**：观察 data 目录结构，理解块文件

### 第 11 课：数据写入流程
**目标**：掌握数据从抓取到存储的完整路径
- 抓取 → 解析 → 存储
- 样本（Sample）结构
- Fanout Storage 机制
- 远程写入（Remote Write）
- **实践**：配置 Remote Write，观察数据流向

### 第 12 课：数据查询流程
**目标**：掌握查询从 API 到存储的完整路径
- HTTP API → PromQL Engine → Storage
- 查询类型（Instant、Range）
- 本地与远程数据合并
- 惰性执行机制
- **实践**：使用 API 执行查询，分析响应

## 第三阶段：核心模块深入（第 13-20 课）

### 第 13 课：TSDB 深入 - Head Block
**目标**：理解内存 Head Block 的实现
- Head Block 的数据结构
- 时间线（Time Series）管理
- 内存映射（mmap）技术
- 检查点（Checkpoint）机制
- **阅读**：TSDB Head Block 博客文章

### 第 14 课：TSDB 深入 - WAL 与恢复
**目标**：理解预写日志的实现
- WAL 文件格式
- 写入与读取流程
- 崩溃恢复机制
- WAL 压缩与清理
- **阅读**：WAL and Checkpoint 博客文章

### 第 15 课：TSDB 深入 - 持久化 Block
**目标**：理解持久化块的存储结构
- Block 目录结构（chunks、index、meta.json）
- 数据压缩算法（Gorilla 编码）
- 索引结构（Inverted Index）
- 墓碑（Tombstones）与删除
- **阅读**：Persistent Block 博客文章

### 第 16 课：TSDB 深入 - 压缩与保留
**目标**：理解数据压缩和保留策略
- 块压缩（Compaction）流程
- 保留策略（Retention）
- 数据过期处理
- 性能优化
- **阅读**：Compaction and Retention 博客文章

### 第 17 课：PromQL 引擎 - 解析器
**目标**：理解 PromQL 的解析过程
- 词法分析（Lexer）
- 语法分析（Parser）
- 抽象语法树（AST）
- 生成解析器流程
- **阅读**：promql/parser 目录源码

### 第 18 课：PromQL 引擎 - 执行器
**目标**：理解 PromQL 的执行机制
- 查询计划生成
- 向量化执行
- 操作符实现
- 函数实现
- **阅读**：promql/engine.go 源码

### 第 19 课：告警与规则管理
**目标**：掌握告警和记录规则的实现
- Rule Manager 架构
- 告警规则评估流程
- 记录规则预计算
- 告警状态管理（Pending、Firing、Resolved）
- **实践**：编写告警规则和记录规则

### 第 20 课：告警通知（Notifier）
**目标**：理解告警发送机制
- Notifier Manager 架构
- Alertmanager 集成
- 告警队列管理
- 服务发现 Alertmanager
- **实践**：配置 Alertmanager，测试告警通知

## 第四阶段：Java 项目集成（第 21-24 课）

### 第 21 课：Java 监控指标基础
**目标**：理解如何在 Java 应用中暴露指标
- Micrometer 库介绍
- Prometheus Java Client
- 指标类型映射（Counter、Gauge、Timer、DistributionSummary）
- 端点暴露（/actuator/prometheus）
- **实践**：在 Spring Boot 项目中集成 Micrometer

### 第 22 课：自定义业务指标
**目标**：学会在 Java 项目中添加自定义监控指标
- 业务指标设计原则
- 创建自定义指标（Counter、Gauge、Histogram）
- 指标命名规范
- 标签设计最佳实践
- **实践**：为业务方法添加性能监控指标

### 第 23 课：配置 Prometheus 抓取 Java 应用
**目标**：掌握将 Java 应用接入 Prometheus 的方法
- 配置抓取任务
- 服务发现配置（Kubernetes、Consul 等）
- 重标签配置
- 多环境管理
- **实践**：配置 Prometheus 抓取多个 Java 应用

### 第 24 课：告警与可视化
**目标**：为 Java 应用配置告警和仪表盘
- 编写 JVM 相关告警规则
- 业务指标告警
- Grafana 仪表盘配置
- 告警通知集成
- **实践**：创建 JVM 监控仪表盘和告警规则

### 第 25 课：实现简易抓取器（Scraper）

- **实践**：实现支持基本查询的引擎

### 第 28 课：实现 HTTP API 与 UI

### 第 29 课：实现告警规则引擎

- 规则配置解析
- 定时评估调度
- 告警状态管理
- 通知发送（Webhook）

### 第 30 课：系统集成与优化
**目标**：整合所有组件，优化性能
- 组件集成与协调

- **实践**：整合所有模块，形成完整系统

5. `promql/engine.go` - 查询引擎
6. `rules/manager.go` - 规则管理
7. `web/web.go` - Web 服务

### 实践项目
1. 配置 Prometheus 监控自身和示例应用
2. 在 Java 项目中添加自定义指标并接入

## 学习建议

1. **循序渐进**：不要跳过基础直接看源码
2. **实践为主**：每节课都要动手实践
3. **阅读测试**：测试是最好的文档
4. **做笔记**：记录关键理解和疑问
5. **分享知识**：写博客或做分享加深理解
