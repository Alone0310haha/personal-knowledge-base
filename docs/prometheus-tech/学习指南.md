# Prometheus 源码学习指南

本指南旨在帮助你系统地学习 Prometheus 监控系统的架构设计与源码实现。

## 一、项目概述

Prometheus 是一个开源的系统和服务监控系统，由 CNCF 托管。其核心特性包括：

- **多维度数据模型**：通过指标名称和键值对标签定义时间序列
- **PromQL**：强大的查询语言
- **Pull 模型**：通过 HTTP 拉取方式收集指标
- **服务发现**：支持静态配置或动态服务发现
- **本地存储**：高效的时序数据库 (TSDB)
- **告警与规则**：支持告警规则和记录规则

### 阶段 2：架构理解（2-3 周）

#### 2.1 内部架构

**核心文档**：
- [documentation/internal_architecture.md](./documentation/internal_architecture.md) - **必读**

**核心组件**：

```
┌─────────────────────────────────────────────────────────┐
│                    Prometheus Server                      │
├─────────────────────────────────────────────────────────┤
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  │
│  │   Retrieval  │  │    Storage   │  │    HTTP      │  │
│  │              │  │              │  │   Server     │  │
│  │  - SD Disc.  │  │  - TSDB      │  │  - UI        │  │
│  │  - Scrape    │  │  - Remote    │  │  - API       │  │
│  │  - Manager   │  │    Read/Write│  │              │  │
│  └──────────────┘  └──────────────┘  └──────────────┘  │
│  ┌──────────────┐  ┌──────────────┐                     │
│  │     Rule     │  │   Notifier   │                     │
│  │    Manager   │  │   Manager    │                     │
│  └──────────────┘  └──────────────┘                     │
└─────────────────────────────────────────────────────────┘
```

**组件说明**：

1. **Scrape Discovery Manager**
   - 服务发现管理
   - 持续更新监控目标列表
   - 源码：`discovery/manager.go`

2. **Scrape Manager**
   - 执行实际的目标抓取
   - 应用标签和重标签配置
   - 源码：`scrape/manager.go`

3. **Storage**
   - 本地 TSDB 存储
   - 远程存储集成
   - 源码：`storage/`, `tsdb/`

4. **PromQL Engine**
   - 查询语言引擎
   - 表达式解析与执行
   - 源码：`promql/engine.go`

5. **Rule Manager**
   - 告警和记录规则评估
   - 源码：`rules/manager.go`

6. **Notifier**
   - 告警发送到 Alertmanager
   - 源码：`notifier/notifier.go`

7. **Web UI & API**
   - Web 界面和 API 服务
   - 源码：`web/web.go`

#### 2.2 数据流理解

**写入流程**：
```
Target → Scrape → Storage (TSDB) → Remote Write
```

**查询流程**：
```
HTTP API → PromQL Engine → Storage (Local + Remote) → Response
```

**告警流程**：
```
Rule Manager → PromQL → Notifier → Alertmanager
```

## 六、常见问题

### Q1: 如何快速定位某个功能的代码？

**A**: 
1. 使用 `internal_architecture.md` 了解组件位置
2. 使用 GitHub 代码搜索
3. 查看相关测试文件

### Q2: TSDB 块是如何组织的？

**A**: 
- 每 2 小时一个块
- 块包含 chunks（数据）、index（索引）、meta.json（元数据）
- WAL 保证崩溃恢复
- 后台压缩成更大的块

### Q3: PromQL 查询是如何执行的？

**A**: 
1. 词法/语法分析生成 AST
2. 创建可执行查询
3. 从存储获取迭代器
4. 惰性执行表达式
5. 返回结果

### Q4: 如何参与贡献？

**A**: 
1. 阅读 CONTRIBUTING.md
2. 选择适合的 issue
3. 遵循 PR 规范
4. 确保测试通过

## 七、学习建议

1. **循序渐进**：不要一开始就深入源码，先理解概念和使用
2. **实践为主**：多动手配置、查询、调试
3. **阅读博客**：TSDB 系列博客质量极高
4. **查看测试**：测试是最好的文档
5. **参与社区**：邮件列表和 IRC 是获取帮助的好地方
6. **做笔记**：记录关键理解和疑问
7. **分享知识**：写博客或做分享加深理解
