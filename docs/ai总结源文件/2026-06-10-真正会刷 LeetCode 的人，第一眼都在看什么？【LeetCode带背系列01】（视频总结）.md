# 真正会刷 LeetCode 的人，第一眼都在看什么？【LeetCode带背系列01】（视频总结）

这期内容不是单题题解，而是一套面向校招、实习、考研复试场景的刷题总纲。作者想纠正一个常见误区：LeetCode 不是让你死背答案，而是让你背题型、背套路、背复杂度边界、背触发条件，最后形成可迁移的解题反应。

[真正会刷 LeetCode 的人，第一眼都在看什么？【LeetCode带背系列01】](https://www.bilibili.com/video/BV1G4oSBMETC)

## 1. 这期内容主要写给谁

- 校招生、实习生、准备考研复试的人：视频明确说，本系列主要服务于需要准备算法题面试的人群。
- 已经具备基础代码能力的人：作者强调，观众至少要理解哈希表、栈、队列等基础数据结构，否则后续内容会跟不上。
- 想提升“面试刷题效率”而不是单纯“多做题”的人：适合已经开始刷 LeetCode，但做题时仍然靠感觉、缺少系统框架的人。

## 2. 讨论角度与要解决的问题

视频是从“面试导向的高效刷题方法论”出发讨论的，核心不是教你某一道题怎么做，而是教你刷题时第一眼应该看什么，刷题前、中、后分别该准备什么、关注什么、沉淀什么。

它主要要解决 3 个问题：

- 为什么很多人刷了很多题，面试里还是不会做：因为只记答案，没有记题型、套路和边界。
- 为什么 LeetCode 能做，笔试和手撕题却写不出来：因为缺少 ACM 模式输入输出能力，以及缺少面试场景下的模板化表达。
- 为什么刷题效率低：因为做题时没有先看数据范围、关键词、目标复杂度，导致方法选择混乱。

## 3. 核心论点与分论点

### 3.1 核心论点：真正会刷题的人，先看“题型、复杂度、关键词、目标”，而不是直接想答案

作者反复强调，刷 LeetCode 不是背代码本身，而是背下面这些更稳定、可迁移的东西：

- 题型：这题本质属于哪一类问题，例如动态规划、二分、双指针、滑动窗口。
- 数据结构与算法：这题常用什么工具来解，例如哈希表、队列、单调栈。
- 时间复杂度边界：在给定数据范围下，哪些方法根本不可用。
- 题目模式：这类题真正需要记住的“触发信号”和“标准解法骨架”是什么。

视频把这种思路先压缩成“四个问题”：

1. 这道题属于什么类型？
2. 应该用什么数据结构和算法？
3. 它允许的时间复杂度大概是多少？
4. 这道题真正需要记住的模式是什么？

这四问是整期视频最核心的总纲，后面的“三补、三看、三背”其实都是围绕这四问展开。

### 3.2 分论点一：刷题前要做“三补”

“三补”指的是，真正开始刷题前，要先把必要能力补齐，否则后面的训练很容易流于表面。

### 3.2.1 第一补：补 ACM 模式能力和最基础代码能力

作者指出，LeetCode 平台大多是函数题模式，平台已经帮你定义好了函数签名，很多时候链表节点、树节点也直接给好了；但真实笔试和很多面试手撕题并不是这样，往往是 ACM 模式。

#### 概念：ACM 模式

ACM 模式指的是题目只给输入输出要求，需要你自己：

1. 写输入输出逻辑；
2. 引入需要的头文件或库；
3. 定义必要的数据结构；
4. 自己组织主程序和函数调用。

这和 LeetCode 模式的差别在于，LeetCode 更像“补全函数”，ACM 更像“从零写完整程序”。

作者给出的准备重点包括：

- 常用输入输出模板；
- 常见头文件或库的引入方式；
- 常用节点定义，例如链表节点、二叉树节点；
- 最基础的数据结构概念。

结论是：如果连输入输出和基本结构都写不出来，哪怕题型记得很多，面试里也很难发挥出来。

### 3.2.2 第二补：补经典题和通用逻辑

作者认为，有些题第一次见会觉得难，但看过题解后会发现它们的套路非常固定，这类题与其反复硬想，不如直接当作“经典模板题”背下来。

视频举了几个典型例子：

- 两数之和：核心是“哈希表查找已出现值”的经典模板。
- 接雨水：核心套路基本固定在“单调栈”或“双指针”。
- LRU 缓存：核心套路基本固定为“双向链表 + 哈希表”。

#### 概念：经典题

这里的“经典题”不是指题目有名，而是指它们承担了某种标准解法的“代表题”角色。你看到题目时，应该能迅速联想到对应的数据结构组合或解法模板。

#### 概念：通用逻辑

通用逻辑指的是跨很多题都会反复出现的思维框架，例如：

- 出现过/是否存在过某个值，优先想到哈希表；
- 两端向中间逼近，优先想到双指针；
- 维护最近最常用、最久未使用，优先想到“链表维护顺序 + 哈希表加速查找”。

### 3.2.3 第三补：补岗位面试信息和目标岗位常考题

作者强调，刷题不能脱离岗位。不同岗位、不同公司，常考题分布并不一样，所以除了刷通用题库，还要做“岗位定向准备”。

他建议重点关注：

- 目标公司的面经题；
- 目标岗位的常考手撕题；
- 同岗位求职者分享的高频题。

#### 概念：面经题

面经题是别人真实面试后总结出来的题目和问题，价值在于它更接近真实场景，不只是平台上的通用题单。

#### 概念：手撕题

手撕题是面试官现场要求候选人写出的算法题或代码题，重点不只是思路，还包括：

- 输入输出组织；
- 代码书写完整度；
- 边界处理；
- 表达过程。

作者的意思是：刷题不应该只对着 Hot100 闷头刷，还要对齐目标岗位的真实考法。

### 3.3 分论点二：刷题时要做“三看”

“三看”是视频最有实操价值的部分，讲的是拿到一道题之后，第一时间应该看哪三类信息。

### 3.3.1 第一看：看数据范围与时间复杂度

作者认为，这是很多人最容易忽视但最关键的一步。

#### 概念：数据范围

数据范围就是题目给你的输入规模，例如 `n <= 20`、`n <= 10^5`、`n <= 10^9`。它直接决定了哪些算法可以用，哪些算法一定会超时。

#### 概念：时间复杂度

时间复杂度是算法随输入规模增长的耗时趋势，例如：

- `O(n)`
- `O(n log n)`
- `O(n^2)`
- `O(2^n)`

作者给出的核心判断逻辑是：

- 如果题目通常要求 1 秒左右完成，经验上总操作量大致不能明显超过 `10^8`。
- 如果 `n = 10^5`，那 `O(n^2)` 大概率会超时，因为已经接近 `10^10` 级别。
- 如果 `n` 很小，例如小于 20，回溯、状态压缩、暴力搜索等方法反而可能可行。

这一步的意义在于：先用复杂度筛掉不可能的方法，再去想具体解法。

### 3.3.2 第二看：看关键词

作者认为，很多题目会在题面中给出明显的“提示词”，这些词其实在暗示你可能使用的模板。

视频里提到的关键词与常见映射关系包括：

- “某个值是否出现过”：
  暗示可以考虑哈希表。
- “连续子串 / 连续子数组”：
  暗示可以考虑滑动窗口，或者前缀和一类方法。
- “两端同时移动”：
  暗示可以考虑双指针。
- “枚举所有方案 / 所有路径 / 所有选择”：
  暗示可以考虑 DFS、BFS、回溯。
- “选或不选 / 最优收益 / 最优方案”：
  暗示可以考虑动态规划，有时也要比较贪心是否成立。

#### 概念：滑动窗口

滑动窗口是一种在连续区间上动态维护状态的方法。通常通过移动左右边界，在 `O(n)` 或接近线性的时间里解决连续子数组、连续子串相关问题。

#### 概念：前缀和

前缀和是把前面若干项的累计结果提前算好，从而快速得到任意区间和或某些区间性质的技巧。

#### 概念：DFS / BFS / 回溯

- DFS：深度优先搜索，适合一条路先走到底再回退。
- BFS：广度优先搜索，适合一层层扩展，常用于最短步数、层序遍历。
- 回溯：本质上是带约束的 DFS，核心是“试探、撤销、再试下一种可能”。

#### 概念：动态规划（DP）

动态规划用于解决“有重复子问题、可以从较小状态推到较大状态”的问题。视频里强调的触发信号是“选或不选”和“最优解”。

### 3.3.3 第三看：看题目的目标要求

作者特别提醒，不要忽略题目里直接写出来的复杂度目标。

例如“寻找两个正序数组的中位数”这道题，题目明确要求时间复杂度应为 `O(log(m+n))` 量级。作者认为，这不是可有可无的提示，而是在直接告诉你：

- 面试官希望你用某种更优的方法；
- 普通合并数组再求中位数的思路虽然能做，但不符合这道题的考点；
- 这种复杂度要求本身就在指向二分法一类的解法。

#### 概念：看目标

“看目标”指的不是看题目想让你输出什么，而是看题目对解法质量的要求，例如：

- 最优时间复杂度；
- 最优空间复杂度；
- 是否必须原地修改；
- 是否要求线性时间。

作者的核心观点是：这类约束就是考点，不能当作装饰。

### 3.4 分论点三：刷题后要做“三背”

视频的 ASR 文稿里有“三倍”的表述，但结合上下文，作者实际想表达的是“三背”，也就是刷完题之后真正要背下来的内容。

### 3.4.1 第一背：背题型模板与触发条件

作者认为，LeetCode 题面会把信息给得很全，但真实面试不一定这样。面试官往往只给你一个简化描述和少量样例，这时候你必须靠平时积累的“题型识别能力”快速定位套路。

#### 概念：题型模板

题型模板是某类题的标准解法骨架，例如：

- 二分法怎么取中点、怎么收缩边界；
- BFS 怎么维护队列；
- 滑动窗口什么时候扩张右边界、什么时候收缩左边界；
- DP 状态如何定义、状态转移如何写；
- LRU 中哈希表负责什么、双向链表负责什么。

#### 概念：触发条件

触发条件就是你一看到某些题面特征，就应该条件反射般联想到某类模板。例如：

- “下一个更大元素 / 下一个更小元素”：
  容易触发单调栈。
- “选或不选、最大收益、最优方案”：
  容易触发动态规划。
- “连续子数组”：
  容易触发滑动窗口或前缀和。

### 3.4.2 第二背：背模板里的关键动作，而不是只背结果

作者并不主张死背整段代码，而是强调要背“模板里的关键动作”。例如：

- 二分法如何更新左右边界；
- BFS 什么时候入队、什么时候出队；
- 滑动窗口什么时候右扩、什么时候左缩；
- 动态规划是一维还是二维，转移方程怎么确定；
- LRU 什么时候查哈希表，什么时候移动或删除双向链表节点。

也就是说，真正需要形成肌肉记忆的是“结构和动作”，这样题目稍微变化，也还能迁移。

### 3.4.3 第三背：用 AI 辅助整理和复习

视频最后提到，AI 已经非常普及，可以拿来辅助刷题后的整理与记忆，例如：

- 帮你提炼题型；
- 帮你总结关键词到模板的映射；
- 帮你把题目的核心结论压缩成可复习的笔记。

这里 AI 的角色不是替你做题，而是辅助你做归纳、压缩和复盘。

## 4. 这套方法具体怎么操作

为了不混场景，下面严格按视频原顺序，拆成“刷题前、刷题中、刷题后”三段操作。

### 4.1 场景 A：刷题前怎么做

1. 确定目标场景：校招、实习、考研复试，或者某个具体岗位。
2. 补齐基础代码能力：确保自己知道哈希表、栈、队列等基础结构在做什么。
3. 补齐 ACM 模式能力：准备输入输出模板、常用头文件、常见节点定义。
4. 整理经典模板题：例如两数之和、接雨水、LRU 这类高频代表题。
5. 收集目标岗位的面经题和手撕题：让刷题内容更贴近真实面试。

### 4.2 场景 B：刷题时怎么做

1. 先看数据范围：先判断 `O(n)`、`O(n log n)`、`O(n^2)` 哪些可能成立。
2. 再看关键词：识别连续子数组、是否出现过、两端移动、选或不选等信号。
3. 再看题目目标：是否明确要求某个时间复杂度或空间复杂度。
4. 根据前三步缩小候选方法：例如哈希表、双指针、单调栈、DP、二分、回溯。
5. 再进入具体求解：此时你是在“有限候选集”里做选择，而不是完全裸想。

### 4.3 场景 C：刷题后怎么做

1. 复盘这道题属于什么题型。
2. 记录它的触发条件：题面里哪些词、哪些约束让你应该想到这种方法。
3. 记录模板骨架：关键步骤是什么，边界如何处理。
4. 提炼成可背诵的短笔记：例如“下一个更大元素 -> 单调栈”“最优收益 + 选不选 -> DP”。
5. 用 AI 或自己的笔记系统做二次压缩，形成后续复习卡片。

## 5. 实际应用时的启示

### 5.1 启示一：刷题效率不取决于做了多少题，而取决于是否建立了“识别系统”

真正的差异不在于谁见过更多答案，而在于谁能快速判断：

- 这题大概是什么类型；
- 哪些方法一开始就该排除；
- 关键词在暗示什么套路；
- 面试官真正想考什么能力。

### 5.2 启示二：LeetCode 成绩不等于面试表现

如果只会在平台给定函数里补代码，到了需要自己写输入输出、自己定义结构、自己解释思路的场景，表现可能会明显掉档。所以“平台刷题能力”和“真实手撕能力”必须同时准备。

### 5.3 启示三：要把刷题和岗位绑定，而不是把刷题当成纯竞赛

作者多次强调岗位差异，这意味着更高效的做法不是无差别刷题，而是：

- 用通用题型打底；
- 用目标岗位高频题做重点突破；
- 用真实面经校准训练方向。

## 6. 可直接套用的 SOP

下面给出一版适合校招/实习算法准备的 SOP，同样按“前中后”顺序组织，不混阶段。

### 6.1 阶段一：刷题前 SOP

1. 先定目标岗位：后端、前端、算法、测试开发等。
2. 列出对应高频来源：Hot100、目标公司面经、同岗位手撕题。
3. 补齐基础模板：ACM 输入输出、常用头文件、链表/树节点定义。
4. 准备经典代表题清单：两数之和、接雨水、LRU 等。
5. 形成自己的“题型目录”：哈希、双指针、滑窗、单调栈、二分、DFS/BFS、DP。

### 6.2 阶段二：刷题中 SOP

1. 拿到题目先看数据范围。
2. 根据数据范围排除不可能的复杂度。
3. 再看关键词，匹配常见模板。
4. 再看题目是否显式要求更优复杂度或更优空间复杂度。
5. 确认候选方法后再写代码。
6. 写完后检查边界条件、空输入、下标越界、重复元素等问题。

### 6.3 阶段三：刷题后 SOP

1. 给每道题打标签：题型、数据结构、复杂度级别。
2. 写一句“触发条件总结”：看到什么特征就想到什么模板。
3. 写一段“模板骨架总结”：这类题解法的关键步骤是什么。
4. 记录容易错的边界：例如二分边界更新、窗口收缩时机、DP 初始值。
5. 每周回顾自己的模板库，而不是只重刷旧题。
6. 用 AI 辅助压缩成更短的复习卡片，方便模拟面试前速刷。

## 7. 一句话总结

这期视频的本质是在讲一套“面试导向的刷题框架”：刷题前补能力，刷题中看边界、看关键词、看目标，刷题后背模板、背触发条件、背关键动作，最终把“会做题”升级为“会识别题、会迁移题、会在面试里稳定写出来”。
