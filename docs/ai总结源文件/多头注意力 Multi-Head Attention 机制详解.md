# 多头注意力 Multi-Head Attention 机制详解

## 来源

- [多头注意力 Multi-Head Attention 机制详解](https://www.bilibili.com/video/BV1of6cBAEuJ/)

## 讨论视角

- 用“人读句子时会动态关注不同信息”的直觉类比，解释注意力机制的目的：在上下文里为“词与词的关系”打分并融合信息
- 从 Transformer 的数据流出发，把注意力拆成可操作的矩阵步骤：Embedding → 线性变换 → 相似度打分 → Mask → Softmax → 加权求和
- 用“把同一份信息按不同关注点分门别类”的思路解释 Multi-Head：并行做多次 attention，每个 head 学到不同的关注维度

## 核心论点与分论点

### 核心论点

- 注意力机制的本质是：为每个词生成“向其他词取信息的权重分布”，再用这个权重把上下文信息加权融合成新的表示；多头注意力就是把这一过程并行做多次，让模型同时从多个关注点建模关系。

### 分论点 1：Embedding 把“词”变成“概念特征集合”，注意力在特征上做关系建模

- 人理解一个字并非只看到字形，而是会联想出大量标签/概念（语义、语法、经验等）
- 大模型用 embedding 将 token 映射为高维向量（作者用“标签打分”类比），后续计算处理的是向量而非原始文字
- embedding 维度很大（作者举例：GPT-2 约 1600 维；DeepSeek 约 7160 维；不少商业模型可能上万维），直接在全维空间做关系计算成本高且信息混杂

### 分论点 2：单头 attention 可以理解成三件事：提特征、算关联、做融合

- 提特征：通过线性层从 embedding 中抽取某一“视角”的特征（作者以“语法特征”为例）
- 算关联：为每个词生成对其他词的“需求/关注”向量，并与对方的“特征/画像”向量做乘法得到相关性分数（矩阵乘法一次算完所有两两关联）
- 做融合：把相关性分数转成可解释的权重（Softmax 归一化为百分比），再用权重对“内容向量”做加权求和得到新的上下文表示

### 分论点 3：Mask 是自回归语言模型的关键约束：只能看见当前位置之前的信息

- 训练/生成是按 token 逐个推进的
- 关联度矩阵右上角（“未来位置”）被置为负无穷，Softmax 后对应权重为 0，确保模型不会利用未来信息

### 分论点 4：多头注意力 = 多个不同视角的 attention 并行 + 拼接融合

- 通过多组不同的线性变换，让每个 head 学到不同的关注点（作者举例：语法版、语义/知识版、感受版等）
- 每个 head 产生一份上下文表示，最后按特征维度拼接成整体输出
- 该输出会作为下一层（下一次 multi-head attention）的输入，重复几十次甚至上百次，最终在多层堆叠中形成强表达能力

## 关键数据 / 案例 / 结论

- 案例（贯穿全片的类比句）：用“我用读读毒蛇”解释同词多义与上下文依赖，说明“注意力焦点会随位置变化”
- embedding 维度示例：
  - GPT-2：约 1600 维
  - DeepSeek：约 7160 维
  - 一些商业模型：可能上万维
- 归一化与数值稳定性：
  - 相关性分数经过 Softmax 变成每行和为 100% 的权重分布
  - 工程上常在 Softmax 前除以 \(\sqrt{d_k}\)（作者以“除以根号下标签数量”类比）以控制分数尺度
  - 作者提示演示分数被刻意放大；实际分数常在 -1 到 1 的小数范围
- QKV 对应关系（作者给出的映射）：
  - K（Key）：抽取出的“特征/画像”（例：语法矩阵）
  - Q（Query）：每个词对特征的“需求/关注”（例：需求矩阵）
  - Softmax\((QK^\top/\sqrt{d_k})\)：关联度/权重矩阵
  - V（Value）：降维后的“内容向量”（作者称为内容矩阵）
  - 权重矩阵 × V：单个 attention head 的输出

## 思维导图（Markmap）

```markmap
# 多头注意力 Multi-Head Attention
## 目标与直觉
### 人理解语言=随上下文动态关注不同信息
### 模型也需要：刻画词与词的关联，并融合上下文
## 原料：Embedding
### token -> 高维向量（概念特征集合）
### 维度很大：GPT-2~1600，DeepSeek~7160，商业模型可能上万
### 信息混杂 -> 需要分视角处理
## 单头 Attention 的流程（矩阵视角）
### 1) 线性变换抽取特征
#### 从 embedding 抽取某视角特征（例：语法）
### 2) 计算关联度（打分）
#### 为每个词生成 Query（需求/关注）
#### 为每个词生成 Key（特征/画像）
#### 相似度：QK^T（矩阵乘法一次算完）
### 3) Mask（自回归约束）
#### 未来位置置为 -inf
#### Softmax 后权重为 0
### 4) Softmax（归一化成权重）
#### 每行变成概率分布（和=1）
#### 规模控制：除以 sqrt(d_k)
### 5) 加权融合
#### Value：对 embedding 再线性变换降维得到 V（内容）
#### 输出：Softmax(QK^T/sqrt(d_k)) * V
## 多头（Multi-Head）
### 多组独立的 Q/K/V 投影并行
### 每个 head 学到不同关注点
### 拼接各 head 输出 -> 总输出
### 堆叠多层（几十/上百次）-> 表达能力提升
## QKV 公式对应
### Q=Query，K=Key，V=Value
### 权重=Softmax(QK^T/sqrt(d_k))
### 输出=权重*V
```
