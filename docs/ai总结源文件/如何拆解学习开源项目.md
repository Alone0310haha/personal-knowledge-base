## 你将得到什么

用一套“从大到小、逐层递进”的流程，快速拆解并学会任意开源项目：先用 DeepWiki 获取项目全景，再用 Claude Code 生成一套可执行的学习课程，并在学习过程中把不懂的点无限下钻。

## 操作清单（照着做）

### 1) 准备学习目标与工作区

1. 选定你要学习的开源项目（GitHub 仓库链接）。
2. 在本地为该项目建一个学习工作区：
   - 克隆仓库到本地
   - 在仓库根目录新建一个用于存放学习资料的文件夹（例如 `tutorials/` 或 `docs/learn/`）

### 2) 用 DeepWiki 做项目“全景扫描”

1. 打开 DeepWiki，把 GitHub 仓库地址导入，让它生成结构化概览页面。
2. 如果概览是英文：
   - 安装浏览器插件“沉浸式翻译”
   - 开启“中英对照”模式，中文读不顺时对照英文原文确认术语
3. 至少通读一遍概览内容，目标是回答：
   - 这个项目解决什么问题
   - 核心模块/目录结构是什么
   - 关键入口文件和关键流程在哪里

### 3) 把概览“落地”为可执行的学习路线（先搭框架）

1. 打开 Claude Code（或等价的代码助手）并指向该仓库。
2. 先让它生成“课程大纲”，而不是直接生成所有课程细节：
   - 建议要求覆盖“全部知识点”，并强调“循序渐进”
   - 课时数量可用 30+ 作为参考，也可以按项目规模调大/调小
3. 让它把课程大纲保存成一个 Markdown 文件，放到你建的学习资料目录里（例如 `tutorials/课程大纲.md`）。
4. 用 VS Code 打开该文件，使用 Markdown 预览模式阅读，确认：
   - 课程被拆成了若干模块
   - 每一课都有目标、关键文件（以及建议练习/阅读点）

### 4) 从第 1 课开始逐课生成（再填内容）

1. 按课程大纲，从“第 1 课”开始，让 Claude Code 生成这一课的详细教程内容。
2. 强制要求输出到文件（不要只输出到终端对话里），保存到学习资料目录（例如 `tutorials/01_总览与运行.md`）。
3. 继续生成第 2、3 … 课，保持：
   - 每课只解决一个清晰目标
   - 先能跑起来/先理解最小闭环，再逐步深入模块与源码

### 5) 用 CLAUDE.md 固化“默认指令”，减少重复沟通

1. 在仓库根目录创建 `CLAUDE.md`，写清楚长期不变的规则，例如：
   - 你正在学习该项目
   - 所有教程/学习产物都保存到 `tutorials/`（或你指定的目录）
   - 输出用 Markdown，结构固定（目标、前置、步骤、关键文件、检查点、常见坑）
2. 在后续对话里，让 Claude Code 自动加载/引用 `CLAUDE.md` 的内容，这样你就不需要每次都重复“保存到哪个文件夹”等要求。

### 6) 遇到不懂就“就地挖井”，并把结果沉淀下来

1. 阅读教程或源码时遇到难点，直接问 Claude Code：
   - “这个函数/模块的作用是什么？为什么需要它？它是怎么工作的？”
2. 如果解释里又出现新概念或新函数，继续追问，形成“无限嵌套深入”。
3. 把关键解释也保存为笔记文件（例如 `tutorials/notes_函数名.md`），便于回看与复习。

### 7) 控制成本（可选）

1. 如果直接使用官方模型成本高：
   - 使用 API 中转/代理方案（通常有更低单价）
2. 也可以用国产模型替代，但可能需要在效果和成本之间权衡。

## 工具、材料与注意事项

- 工具
  - DeepWiki：生成项目结构化概览，适合“从大到小”建立地图
  - 沉浸式翻译：中英对照阅读，降低英文文档门槛
  - VS Code：阅读仓库与教程文件，使用 Markdown 预览更舒服
  - Claude Code（或等价工具）：生成课程、逐课讲解、按需深入
- 注意事项
  - 先做“课程大纲”再逐课填充，避免一次性输出过长导致质量下降或中断
  - 每次输出都落到文件里，形成可复用的学习资产
  - 学习的关键不是工具，而是“先全景、后深入、再下钻”的方法

## 常见错误与解决方案

- 错误：上来就硬啃源码，看半天没有整体框架
  - 解决：先用 DeepWiki 通读概览，先建立“地图”再进代码
- 错误：让 AI 一次性生成所有课程，结果内容不完整或质量不稳
  - 解决：先生成课程大纲，再逐课生成与迭代
- 错误：教程输出在对话里，后续找不到/难以复习
  - 解决：每次生成都要求保存为 Markdown 文件，并统一放到固定目录
- 错误：每次都要重复“保存到哪、按什么格式写”
  - 解决：用 `CLAUDE.md` 固化默认要求，让工具自动遵循
- 错误：卡在某个细节点，学不下去
  - 解决：就地追问（可无限下钻），并把解释沉淀到笔记文件
- 错误：使用成本太高
  - 解决：使用 API 中转方案，或选择合适的替代模型
