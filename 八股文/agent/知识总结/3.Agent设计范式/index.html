    3.Agent设计范式

    Agent 的设计范式有哪些？¶

    15秒简答： Agent 设计范式本质是在定义“任务怎么被决策和推进”。核心可以按能力递进理解：CoT 让模型显式思考，ReAct 把思考和工具行动串成循环，Plan-and-Execute 先全局规划再分步执行，Reflection 在执行后做质量检查和修正。实际工程里常把它们组合使用，而不是彼此互斥。

    3分钟详答：
    Agent 的设计范式解决的是“LLM 不只是回答一句话，而是如何持续推进任务”的问题。直接调用 LLM 时，模型通常只根据上下文一次性生成答案；遇到复杂任务时，中间推理、外部信息获取、执行结果校验都容易混在一起，所以需要一套任务推进结构。

    第一层是 CoT，也就是 Chain of Thought。它让模型先把推理过程显式写出来，再给结论。这样做的原因是 LLM 的输出是顺序生成的，前面写出的推理步骤会进入上下文，成为后续生成的依据，从而降低多步推理里隐式状态出错的概率。但 CoT 的局限也很明显：它主要是纯文本推理，不能原生获取实时数据、执行计算或访问外部系统。

    第二层是 ReAct，也就是 Reasoning and Acting。它把 CoT 的“思考”扩展成 Thought -> Action -> Observation 循环：模型先分析当前情况，再决定调用哪个工具，工具返回结果后作为 Observation 写回上下文，模型基于新观察继续思考。注意 ReAct 的循环不是模型自己跑起来的，而是外层代码框架负责解析 Action、执行工具、回填 Observation、再次调用模型。它适合开放问答、信息搜索、客服等需要边探索边决策的任务，优点是灵活、透明，缺点是长任务容易漂移，步骤多了 token 消耗也会上升。

    第三层是 Plan-and-Execute。它针对 ReAct “走一步看一步、长任务容易跑偏”的问题，把规划和执行解耦：先由 Planner 把目标拆成完整步骤，再由 Executor 逐步执行。成熟实现还会做动态 Replan，每执行完一步就根据结果判断剩余计划是否需要调整。它适合深度研究、长文写作、多工具数据分析这类目标明确、步骤依赖明显的复杂任务，代价是多了规划和重规划的 LLM 调用，系统实现也更复杂。

    第四层是 Reflection。它不是一套独立的主流程，而是叠加在 ReAct 或 Plan-and-Execute 上的质量保障机制：执行完一步或整个任务后，让模型检查结果是否达标，不达标就重试或换策略。它适合代码生成、正式报告、法律文书等质量要求高、错误代价大的场景；代价是增加 token 消耗和延迟。Reflection 的变体 Reflexion 还会把失败原因总结成经验，作为下一次尝试的上下文，相当于让 Agent 在推理阶段“记住教训”。

    工程选型可以按一句话记：简单探索用 ReAct，复杂长任务用 Plan-and-Execute，高质量输出叠加 Reflection；CoT 是底层推理增强手段，常作为 ReAct 的思想基础。生产环境里还要注意，纯 Agent 不一定是首选，更常见的是 Agentic Workflow：主流程用 Workflow 保证可控，在需要灵活判断的节点嵌入 Agent 能力。

    追问：CoT 是什么？
    CoT 是让模型先显式写出推理步骤再给答案，本质是把隐式中间状态写进上下文，减少多步推理误差。

    追问：CoT 的局限是什么？
    CoT 主要解决“怎么想”，但不原生解决“怎么和外部世界交互”，所以拿不到实时数据，也不能直接执行工具。

    追问：ReAct 是什么？
    ReAct 是 Thought -> Action -> Observation 的循环，把显式推理和工具调用交织起来，让 Agent 能边想、边做、边根据结果调整。

    追问：ReAct 的循环是谁驱动的？
    不是模型自己循环，而是代码框架驱动：框架解析模型输出、执行工具、写回 Observation，再次调用模型。

    追问：Plan-and-Execute 解决什么问题？
    它解决 ReAct 长任务容易漂移的问题，先全局规划，再分步执行，让复杂任务有清晰结构和目标约束。

    追问：动态 Replan 是什么？
    动态 Replan 是每执行完一步后，把结果反馈给规划器，判断剩余计划是否还合理，必要时更新后续步骤。

    追问：Reflection 是什么？
    Reflection 是质量检查和修正机制，在执行后评估结果是否达标，不达标就重试或调整策略。

    追问：Reflection 和 Reflexion 有什么区别？
    Reflection 偏向检查后重试；Reflexion 会把失败原因和改进建议总结成经验，放进后续上下文。

    追问：ReAct、Plan-and-Execute、Reflection 怎么选？
    任务简单、边界开放用 ReAct；任务长且依赖多用 Plan-and-Execute；结果质量要求高，就在前两者上叠加 Reflection。

    追问：Agent 和 Workflow 的关系是什么？
    Workflow 是开发者写死流程，确定性高；Agent 是 LLM 动态决策，灵活但不可控。生产里常用 Workflow 控主链路，在局部节点嵌入 Agent。
