## 2. 总体原则：按“面试追问路径”来复习（由浅入深）

面试常见追问顺序大概率是：基础语义 → 源码/原理 → 线上问题 → 方案取舍 → 系统设计。

因此复习也按这个顺序推进：

1) 基础与原理先补齐（不怕追问）  
2) 再做中间件与微服务（能讲清“为什么这么设计”）  
3) 最后集中练系统设计与排障表达（能扛住“给场景你怎么做”）

## 3. 复习路径（循序渐进）

下面按“阶段”组织；每个阶段都给出：

- 你需要掌握的知识点
- 典型面试题方向
- 你应该产出的“可验证成果”（避免只看不练）

### 阶段 A：计算机基础与编码基本功（先打底，防止一问就露馅）

**知识点**

- 数据结构与算法：复杂度、数组/链表/栈队列/哈希/树/堆、排序、二分、滑动窗口、双指针、回溯
- 计算机网络：HTTP/HTTPS、TCP 三次握手/四次挥手、长连接、常见超时与重试风险
- Linux：文件权限、进程/线程、常用命令与排障入口

**典型面试题**

- 手写 LRU / TopK / 生产者消费者
- 解释一次 HTTP 请求在服务端的完整链路（含网关/Nginx、连接复用、超时）
- 线上 CPU 飙高、接口 RT 飙高、连接数暴涨，第一时间怎么查

**成果**

- 准备 20 道算法题（覆盖常见模式），每道写出“思路 + 复杂度 + 边界条件”
- 准备 10 条 Linux 排障命令清单（每条写“看什么指标/异常长什么样”）

### 阶段 B：Java 核心（集合/IO/并发/常见坑）

**知识点**

- 基础语义：equals/hashCode、泛型擦除、反射、注解
- 集合：HashMap/ConcurrentHashMap、ArrayList、LinkedList、fail-fast
- IO/NIO：BIO/NIO/AIO、Selector 基本模型、零拷贝概念

**典型面试题**

- HashMap 扩容、冲突解决、为什么线程不安全
- ConcurrentHashMap 的并发控制思路
- NIO 为什么吞吐更好，和线程模型关系是什么

**成果**

- 能把“HashMap vs ConcurrentHashMap”讲成 3 分钟结构化回答（定义→原理→坑→使用建议）
- 写一个小 Demo：线程池 + 队列处理任务，输出可观测指标（成功/失败/耗时分布）

### 阶段 C：并发与线程池（高频、追问深）

**知识点**

- JMM、volatile、synchronized、CAS、AQS
- Lock 与 Condition、并发工具类（CountDownLatch/Semaphore/Concurrent*）
- 线程池参数与策略：队列/拒绝策略/线程回收、如何避免“无界队列/无界线程”

**典型面试题**

- 线程池打满怎么办：如何定位、如何降级、如何限流、如何拆分任务
- 死锁怎么排查（线程 dump 的关键字段怎么看）

**成果**

- 输出一份“线程池事故复盘模板”：现象→指标→定位→根因→修复→预防
- 熟悉一次 jstack 的阅读路径（至少能识别死锁与热点线程）

**知识点**

**典型面试题**

**成果**

## 4. 一条最有效的“贯穿式练习主线”（建议边学边做）

做一个小型但可扩展的业务系统（选你熟悉的：下单/库存/优惠券/支付回调 任一）：

1) Spring Boot + MyBatis + MySQL：先把核心链路做通（含事务、唯一约束、异常处理）  
2) 加 Redis：缓存查询 + 分布式锁/防重（至少覆盖击穿/雪崩/穿透其中 2 个）  
3) 加 MQ：异步解耦（下单后异步发券/发通知/记账），做幂等与重试  
4) 加可观测性：日志规范、关键指标（QPS/RT/错误率）、慢 SQL 记录  
5) 刻意制造故障并修复：慢 SQL、缓存击穿、重复消息、线程池打满、Full GC

面试表达时，用“场景 → 目标 → 方案 → 取舍 → 风险与兜底 → 结果数据”来讲，会显得非常工程化。

## 6. 最后冲刺：高频题清单（你应该能“脱口而出”）

- Java：HashMap/ConcurrentHashMap、线程池、volatile/synchronized、AQS、NIO
- JVM：内存结构、GC、OOM 与排查、常见参数与工具链路
- MySQL：索引与 explain、事务/MVCC/锁、慢查询与分页优化
- Redis：三大缓存问题、分布式锁正确性、热点 Key

- 排障：CPU/内存/RT/连接数异常的第一小时怎么做
